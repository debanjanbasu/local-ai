# Implementation Status

_Code-verified status for the `gemma-4-e2b-qat` path plus packaged multimodal assets._

## Summary

The engine runs the Unsloth Gemma4 E2B QAT model (`gemma-4-E2B-it-qat-UD-Q2_K_XL.gguf`) end-to-end on Metal and **produces correct, coherent text** ("The capital of France is Paris", etc.). The text decoder path is implemented: per-layer head dims (256 SWA / 512 full), proportional NEOX RoPE, KV sharing (layers 15–34 reuse a source layer's cache), per-layer-input (PLE) injection, embedding scaling, QK/V norms, sliding-window masking, and final logit soft-capping. The huge per-layer embedding table (~2.35 B params) is **streamed row-by-row from CPU on demand** rather than dequantized into VRAM, exploiting the E2B "embed-2B" structure (~3.4 GB resident saving); heavy model execution remains Metal GPU + CPU. Defaults are capability-first: max model context when memory allows, model/context-max output, and quality sampling (`temperature=0.7`, `top_p=0.95`). The live text path also has an MLX-style prompt-prefix cache: repeated prompts skip prefill, and shared-prefix prompts restore quantized KV rows before running only the suffix. The runtime preserves Gemma 4 multimodal config/token IDs, packages the downloaded Unsloth `mmproj-BF16.gguf` companion projector in `model.lma`, loads companion modality tensors into device-resident Metal buffers, and `chat --image`/`MediaInput::DecodedImage` execute the GPU SigLIP-style vision path into decoder soft-token prefill. `chat --video`/`MediaInput::DecodedVideo` accept decoded frames, sample up to 32 frames, insert Gemma4VideoProcessor-style timestamped 70-token video spans, run each frame through the same GPU vision/projector path, and reach decoder soft-token merge. `chat --audio`/`MediaInput::PcmAudio` accept PCM16/float32 audio, downmix/resample to Gemma 4's 16 kHz log-mel cadence, run duration-derived audio slot reservation, GPU strided-conv frontend, 12-layer USM/Conformer-style tower, final audio projectors, and decoder soft-token merge. Real-model image and audio smokes reach generation; video frame-input smoke reaches the generation path. `benchmark --multimodal-suite` generates deterministic in-memory decoded image/audio/video fixtures so the executable benchmark covers the same app handoff paths without committing media artifacts. The current task's targeted checks pass.

## Out of scope (removed)

- Qwen MoE model and all MoE/expert kernels
- Mamba/SSM layers and kernels
- IQ mixed-precision quantization kernels
- Qwen-era vision/audio code paths; Gemma 4 E2B image, sampled-frame video, and WAV audio inputs now use the optimized Unsloth companion assets through Gemma-specific GPU towers and decoder soft-token prefill.
- Safetensors loading — the project targets the Unsloth QAT GGUF only (and `.lma` built from it)
